# Backbrief question graph — every question → answers → config effect

> **Data file, not a procedure.** The complete setup decision tree as one machine-readable
> map: every question the plugin ever asks, the possible answers, the exact keys each
> answer writes, what it gates later, and what a skip costs. A cold reader (human or AI)
> can reconstruct every setup outcome from this file alone, without opening the five
> procedure files. Procedures (`start.md`, `profiles.md`, `tasks.md`, `deploy.md`,
> `status.md`) remain the source of truth for *wording and flow*; on any conflict, the
> procedure wins and this file must be updated in the same change.
>
> Write targets used below:
> - `state:<key>` — `.backbrief/state.yaml` via `state.js set` (canonical keys: `state.js --help`)
> - `tenant:<path>` — `tenant.yaml` (schema: `plugin/templates/tenant.schema.json`)
> - `secrets:<VAR>` — `.backbrief/secrets.env` (gitignored, chmod 600; never in git)
> - `waitlist` — `.backbrief/waitlist.yaml` via `state.js waitlist-observe` (+ opt-in gateway POST)
> - `roadmap` — `.backbrief/roadmap.md`, auto-regenerated by `state.js` on every `steps.*`/`fork` write
>
> Budget invariants (`_conventions.md` §3): ≤5 questions to the first artifact (A0);
> later rungs 1 primary ask + ≤2 confirms; corrections and CONFIRMs are free.
> Every fork also accepts the §13 menu (our-stack / skip / own-tool / set-it-up-for-me).

## Rung A0 — survey → vault skeleton (`start.md`) `[REQUIRED — entry rung]`

```yaml
- id: A0.0-surface-gate
  rung: A0
  prompt: (NOT a question) PREFLIGHT surface/Node gate — bind fs.write + run.script first
  answers:
    - script-capable surface (terminal, Node >= 18, filesystem) -> continue to A0.Q1
    - chat-only / no terminal / no Node -> honest exit: point at hosted waitlist, never
      fake a vault; on a typed email -> telemetry.js waitlist --interest=hosted --source-step=A0
  default: continue (most surfaces)
  writes: [waitlist(email optional) on the no-terminal branch]
  gates: [everything — the A0 scripts cannot run on a chat-only surface]

- id: A0.2-inferences
  rung: A0
  prompt: (NOT a question) inferred-and-STATED assumptions — corrections are free
  answers: [conversational corrections only]
  writes:
    - company name (init-vault --company) <- git remote / email domain / cwd
    - languages (init-vault --languages) <- conversation + volunteered team languages
    - state:owner <- who is setting up (git user.name/email or self-reference); B1 matches
      it to the roster and sets tenant:roster[].is_owner=true (fixes the S4 hard-fail)
    - tenant:timezone (init-vault --timezone) <- stated location / locale / session tz;
      never silently UTC — STATE it
  default: assumptions stand until corrected
  gates: [A0.8 init-vault flags; B1 is_owner resolution; filename timestamps use timezone]

- id: A0.Q1
  rung: A0
  prompt: vault location (company name folded in when inference failed)
  answers:
    - Enter                      -> ./<company>-vault
    - any path                   -> that path (warn once if inside an unrelated repo)
  default: ./<company>-vault
  writes: [vault root for all later steps]
  gates: [everything — all files live here]
  notes: company name is INFERRED (git remote / email domain / cwd); on the Claude-chat
    surface (no git, no cwd) infer it from the user's own words, else fold "and what's
    the company called?" into THIS question's sentence — never a separate 6th question.

- id: A0.Q2
  rung: A0
  prompt: persona — whose calls will this vault hold? (headcount folded into free text)
  answers:
    - "1 / mostly mine (solo founder, consultant)"  -> persona=solo
    - "2 / my team's (team lead inside a company)"  -> persona=team_lead
    - "3 / whole company's (company/leadership)"    -> persona=company_lead
    - volunteered headcount ("3 — about 40 people") -> team_size_bucket=lt10|10-50|gt50
  default: "3 (company_lead); team_size_bucket=lt10 when no headcount volunteered"
  writes: [state:persona, state:team_size_bucket, tenant:tenant.persona (at A0.8), "init-vault --persona"]
  gates:
    - team_size_bucket selects the A0.Q3 variant (golden vs custom)
    - persona shapes A0.Q4 (solo: dropped; team_lead: team-name wording; company_lead: full)
    - persona shapes the skeleton (solo: no team folders — general/ carries everything)
    - persona-aware copy at A2.1 (counterparties) and A4.2 (solo ROI line)
  skip_effect: Enter = company_lead + lt10 (today's full tree)

- id: A0.Q3a
  rung: A0
  prompt: stack fork, team_size_bucket=lt10 (golden path)
  answers:
    - Enter                -> stack={calls: zoom, chat: slack, tracker: linear, git: github}, stack_path=golden
    - "I use X instead"    -> [LLM] parse free text into the same stack map, stack_path=custom
  default: golden path
  writes: [state:stack.calls|chat|tracker|git, state:stack_path]
  gates: &stack-gates
    - stack.tracker -> tenant:features.tracker.kind at B1 (generate-tenant.js)
    - stack.chat != slack -> tenant:features.slack.enabled=false at B1 + B3 opens with recognition, not a pitch
    - stack.calls != zoom -> B2 opens with recognition ("you said Meet — waitlist territory, skip or connect Zoom anyway?")
    - stack.git != github -> vault.repo stays null at B1 + B4 opens with recognition (GitLab = waitlist-only)
    - every unsupported slug -> connector_demand (telemetry) + waitlist (observed) at A0.8
  skip_effect: n/a (defaults apply)

- id: A0.Q3b
  rung: A0
  prompt: stack fork, team_size_bucket>=10 ("what do you use for calls/chat/tracking?")
  answers:
    - free text ("Meet, Slack, Jira") -> [LLM] parse into stack map, stack_path=custom
  default: none (one line of free text; unknown tools are NOT challenged here — intent-first at the rung where they matter)
  writes: [state:stack.*, state:stack_path]
  gates: *stack-gates
  skip_effect: unanswered fields default to the golden tool

- id: A0.Q4
  rung: A0
  prompt: team names (persona-shaped; droppable)
  answers:
    - solo persona            -> QUESTION DROPPED (no team folders; general-only)
    - team_lead + Enter       -> --teams <team> (one named team folder)
    - team_lead + sub-areas   -> --teams <team>,<sub1>,<sub2>
    - company_lead + Enter    -> general-only (no team folders)
    - company_lead + tags     -> --teams <tag,tag> (one folder per tag)
    - already named in Q3 free text -> QUESTION DROPPED (reuse)
  default: general-only (company_lead) / <team> (team_lead) / dropped (solo)
  writes: ["init-vault --teams", tenant:vault.teams (at B1 from folders)]
  gates: [team routing at A1.3, tracker team_mapping at B5]
  skip_effect: everything routes to the mixed folder (general/)

- id: A0.Q5
  rung: A0
  prompt: telemetry opt-in
  answers:
    - Enter / no  -> tenant:features.telemetry.enabled=false (all telemetry.js calls no-op)
    - yes         -> tenant:features.telemetry.enabled=true + install_id generated (UUIDv4, local)
  default: "no — consent never defaults to yes"
  writes: [tenant:features.telemetry.enabled, tenant install_id on consent]
  gates: [every TELEMETRY line in every procedure]
  skip_effect: fully functional, zero pings

- id: A0.9-company
  rung: A0
  prompt: (NOT a question) docs/company.md drafted from A0 facts + inference, every inferred
    line suffixed "(inferred — correct me)"; website inferred from the corp email domain /
    git remote / a volunteered URL -> frontmatter website:, and when the session has web
    access the homepage (+ /about|/product when trivially reachable) is fetched to pre-fill
    what-we-do/products/market — site-sourced lines suffixed "(from site — correct me)".
    No web access -> one degradation line, never a question, never blocks.
  answers: [conversational corrections only — free]
  default: draft stands until A2 enrichment
  writes: ["vault docs/company.md (template: plugin/templates/frontmatter/company-profile.md, <=60 lines, website: field)"]
  gates: [A1.4/A3.1 digest+task context, A2.3 enrichment (optional site re-check), B1 tenant.about fill, glossary seeds]
  skip_effect: n/a
```

Privacy routing is not part of v0.1 (owner decision): no privacy briefing, no sensitivity
questions, no private-route confirms anywhere in this graph. A user asking for private
1:1/board/legal handling -> honest one-liner + `state.js waitlist-observe privacy` (+
`telemetry.js waitlist --interest=privacy` on a typed email).

Zero-questions mode (paste-only user): A0.8 runs with all defaults (`--telemetry no`,
persona=company_lead, golden stack), then jumps to A1.2 with the paste.

## `status` — zero questions

Reads state + scripts, renders the report incl. the **Deferred** line
(count + top entries + `→ .backbrief/roadmap.md`) and the deterministic
**Next** suggestion (first incomplete rung in ladder order:
a0 a1 a2 a3 a4 b0 b1 b2 b3 b4 b5 b5_5 b6 b7 b8).

## Invariant checklist (verify on any flow change)

- Worst case at A0 stays 5 questions (solo persona: 4); statements/CONFIRMs are free.
- Every `writes:` target above exists in `state.js` canonical keys or
  `plugin/templates/tenant.schema.json`.
- Every answer that names an unsupported tool ends in `connector_demand` (telemetry) +
  `waitlist-observe` (local mirror), never a "no" — including tools named at A0/A1 and
  GitLab at B4 (the roadmap-pricing signal must not be under-counted).
- Every skip lands in `.backbrief/roadmap.md` with degradation + re-enable path.
- Stack answers recorded at A0 are consumed at B1 (generate-tenant.js) and recognized at
  the A2, B2/B3/B4 preflights and the A3.2 recorded-tracker path — the installer never
  re-pitches a tool the user already rejected.
